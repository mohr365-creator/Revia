# Software Quality Assurance Plan (SQAP)

Document: REV-AOS-SQAP-001 rev A — DRAFT

## 1. SQA activities

- Audit each life-cycle process against its plan and standards at
  every transition (entry/exit criteria from the SDP).
- Witness verification runs for-credit (DAL A/B requires SQA records
  of test execution per DO-178C Table A-9/A-10 objectives).
- Conformity review of each software baseline before release
  (DO-178C §8.3): all data items present, traced, under CM; all PRs
  dispositioned; build reproducibility demonstrated.

## 2. Independence

SQA reports outside the development organization (program-launch
staffing). DAL A objectives requiring independence are satisfied by
role separation recorded in review/audit records.

## 3. Records

Audit findings, witness logs, and conformity review minutes are CC1
configuration items referenced from the Software Accomplishment
Summary (SAS) at certification.

## 4. Supplier control

IMA platform hardware and any COTS/PDS software enter through the
DO-297 supplier process; none is currently used.
